# myAGENT.md - 공통 개발 원칙 문서

본 문서는 프로젝트 개발 시 AI가 준수해야 할 공통적인 개발 원칙과 가이드라인을 정의합니다.

## 1. 소통 원칙
- 사용자와의 모든 소통(질문, 계획서 작성, 진행 상황 보고 등)은 기본적으로 한국어를 사용합니다.
- 설명은 명확하고 간결하게 핵심 위주로 전달합니다.

## 2. 작업 절차 및 최소 변경 원칙
- 모든 소스 코드 변경 전에 문제 요건을 충분히 파악하고, 관련 코드를 분석합니다.
- 대규모 리팩토링이나 구조 변경은 명시적 요청이 없는 한 절대 금지하며, 기존 기능을 깨트리지 않는 '최소 변경 원칙'을 최우선으로 합니다.
- 수정 사항이 발생하기 전 구현 계획(Implementation Plan)을 작성하여 사용자의 승인을 거친 뒤 작업을 진행합니다.

## 3. 코딩 표준 및 네이밍 가이드
- PEP 8 스타일 가이드를 준수합니다.
- 변수, 함수명은 명확한 의미를 담아 `snake_case`로 작성하며, 클래스명은 `PascalCase`로 작성합니다.
- 비즈니스 로직과 GUI 로직을 철저히 분리합니다. GUI 의존적인 라이브러리(PySide6 등)는 비즈니스 서비스 레이어에서 직접 임포트하지 않도록 설계합니다.

## 4. 파일 접근 및 안전 규칙
- 프로젝트 외부 경로에 파일을 직접 생성하거나 수정하지 않습니다.
- 불필요한 임시 파일은 작업 완료 후 즉시 정리합니다.
- 기존 주석 및 문서화(Docstring)를 훼손하지 않고 유지합니다.

## 5. 수정 후 자체 검증 루프
- 코드 변경 이후에는 반드시 유닛 테스트 및 실행 확인을 통해 수동/자동 검증을 수행합니다.
- 가상 환경(`.venv`) 내의 종속성을 활성화하여 테스트를 실행하며, 모든 테스트가 통과하는지 확인합니다.
